# GRC RU — концепт-демо сайта

Русскоязычная **витрина дизайна и UX** для компании **GRC** (ДжиЭрСи, [1grc.ru](https://www.1grc.ru)): ремонт промышленного оборудования и спецтехники, выездное и стационарное производство.

> **Это не официальный сайт заказчика** и не замена продакшен-версии 1grc.ru. Параллельно у GRC развивается отдельный **американский** сайт; данный репозиторий — демонстрация подхода к промышленному B2B (структура, тексты, вёрстка, код).

## Задача демо

Показать заказчику, что возможно:

- понятное позиционирование за несколько секунд («что ремонтируем, для кого, почему доверять»);
- язык главного механика, а не корпоративный туман;
- два формата работ — **выезд** и **цех**;
- каталог реальных проектов и разбор сильных кейсов;
- B2B-лендинг с промышленной подачей (фото, цифры, CTA).

Контент и формулировки опираются на разбор текущего 1grc.ru (отчёт Site Doctor, файл `Критика сайта.txt` в корне репозитория).

## Структура проекта

```
src/
├── app/                    # Маршруты App Router
│   ├── page.tsx            # Главная
│   ├── vyezd/ ceh/ raboty/ uslugi/
│   ├── globals.css         # Тема, токены, .grc-band
│   └── layout.tsx
├── components/             # UI-блоки (Header, Hero, Cases, …)
└── data/                   # Контент (без CMS)
    ├── site.ts             # Бренд, услуги, процесс, отзывы
    ├── raboty.ts           # 18 проектов, фильтры
    ├── vyezd.ts ceh.ts     # Страницы производств
    ├── uslugi.ts           # Направления услуг
    └── case-details.ts     # Полные кейсы [slug]
```

Ключевые компоненты: `Header`, `SubHeader`, `Hero`, `TrustBar`, `Directions`, `Services`, `Process`, `Cases`, `Reviews`, `Urgency`, `Contact`, `Footer`, `RabotyCatalog`, `CaseDetailView`, `ServiceDirectionPage`.

## Локальный запуск

**Требования:** Node.js 20+, npm.

```bash
git clone https://github.com/zobnin8-ux/grc_rus.git
cd grc_rus
npm install
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

### Windows: ошибка SSL при `npm install`

Если появляется `UNABLE_TO_VERIFY_LEAF_SIGNATURE`:

```powershell
$env:NODE_OPTIONS='--use-system-ca'
npm install
```

Ту же переменную можно задать в профиле PowerShell, если ошибка повторяется.

### Скрипты

| Команда | Действие |
|---------|----------|
| `npm run dev` | Dev-сервер с HMR |
| `npm run build` | Production-сборка |
| `npm run start` | Запуск после `build` |
| `npm run lint` | ESLint (Next.js) |

## Контент и изображения

- Тексты и метаданные — в `src/data/*.ts` (TypeScript, без админки).
- Фото с официального сайта: `https://www.1grc.ru/wp-content/**`.
- Разрешение домена: `next.config.ts` → `images.remotePatterns`.

Чтобы добавить проект в каталог: запись в `src/data/raboty.ts`. Полный кейс — `src/data/case-details.ts` + маршрут в `generateStaticParams` (`raboty/[slug]/page.tsx`).

## Ограничения демо

- Форма заявки **не отправляет** данные (только UI-состояние «принято»).
- Нет CRM, аналитики, i18n, SEO-редиректов, юридических страниц, блога.
- Телефон в шапке ведёт на `tel:` — на десктопе Windows может открыть приложение «Телефон»; основной CTA на форму `#contact`.
- Сравнение «было/стало» с текущим 1grc.ru **намеренно не делалось** (публичное демо).
